virtualenv 个人指南

发表于 2017-05-05 | 分类于 指南集 | | 阅读次数

virtualenv是解决一个机器上多个应用需要的Python版本不一致的问题，virtualenv就是用来为一个应用创建一套“隔离”的Python运行环境,解决了解决了不同应用间多版本的冲突问题。

安装

安装非常简单

使用

使用也非常简单，就是一些简单的命令

使用步骤如下：

  1. 创建目录

  2. 创建一个独立的Python运行环境，命名为myenv

命令virtualenv就可以创建一个独立的Python运行环境，参数–no-site-packages的作用是我们可以得到了一个不带任何第三方包的“干净”的Python运行环境，即已经安装到系统Python环境中的所有第三方包都不会复制过来。

  1. 用source进入该环境,在 myenv 目录下

命令提示符会改变，有个(myenv)前缀，表示当前环境是一个名为myenv的Python环境,你可以下面正常使用pip安装各种第三方包，并运行python命令

  1. 退出myenv环境，使用deactivate命令
